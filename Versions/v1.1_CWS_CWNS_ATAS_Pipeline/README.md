# ATAS v1.1 — CWS/CWNS Speech Timing Analysis Pipeline for Children

ATAS v1.1 is a CWS/CWNS-focused extension of the Automatic Temporal Speech Analysis (ATAS) workflow.

This repository provides notebooks and supporting scripts for analyzing speech timing and pause-related measures in:

- **CWS** — Children Who Stutter
- **CWNS** — Children Who Do Not Stutter

This version includes an updated processing and project organization structure, along with more robust processing for noisier recordings.

---

# Overview

This repository contains a complete notebook-based workflow for:

- Processing clipped `.wav` audio files
- Detecting speech and pause events
- Generating participant-level event CSV files
- Relabelling or correcting detected events
- Computing long and short pause measures
- Combining ATAS-derived measures with demographic and behavioral variables
- Running group-level statistical analyses
- Generating figures
- Running machine learning classification models

The workflow is organized around one central project directory:

```text
ATAS_CWNS_CWS_Project/
```

Inside each notebook, this directory is referred to as:

```python
base_dir = '.../ATAS_CWNS_CWS_Project'
```

Users should update `base_dir` to match the location of the project folder on their own computer.

---

# Important Data Availability Notice

**Raw audio recordings are not included in this repository.**

This repository may include notebooks, scripts, and derived CSV files, but it does not include the original raw audio recordings.

Depending on the files available in your local copy, this means:

- You can reproduce analyses that start from the derived CSV files.
- You can reproduce figures and statistical models if `Stat_csv_files/CWNS_CWS_all_details.csv` is available.
- You can rerun the full audio-processing pipeline only if you have compatible clipped `.wav` files and the required metadata CSV files.

# Folder Descriptions

## `Audio_files_clipped/`

This folder stores clipped `.wav` files used by the automated ATAS processing pipeline.

These files are expected to match the filenames listed in the group metadata CSV files, such as:

```text
Stat_csv_files/CWNS_final.csv
Stat_csv_files/CWS_final.csv
```

The automated processing notebook checks which `.wav` files are present in this folder and processes only files that are also listed in the relevant metadata CSV.

---

## `Visualize/`

This folder stores `.wav` files used for waveform visualization and manual inspection.

The visualization notebook reads audio from this folder when plotting and checking speech/pause segmentation results.

---

## `Stat_csv_files/`

This is the main CSV directory.

It stores:

- Input metadata CSV files
- Group-level ATAS summary files
- Demographic and behavioral score files
- Combined analysis datasets
- Statistical summary CSV files

The most important final dataset is:

```text
Stat_csv_files/CWNS_CWS_all_details.csv
```

This file is used by the main statistical, visualization, supplementary, and machine learning notebooks.

---

## `Individual_OutputCSV_Files/`

This folder stores individual participant-level event CSV files generated by the automated ATAS processing pipeline.

For example, if the input audio file is:

```text
P001.wav
```

the corresponding event-level output file may be:

```text
P001_50_150.csv
```

These files contain event-level speech and pause information and are used for event relabelling and downstream pause analyses.

---

This folder contains function notebooks used by the main analysis notebooks.

Current function notebooks:

```text
Long_short_pause_compute_functions.ipynb
```

These notebooks are not primarily intended as standalone analyses. They define helper functions used by the main processing and pause-computation notebooks.

## `Notebooks/ATAS_Visualization/`

Contains the visualization notebook:

```text
CWS_and_CWNS_ATAS_Visualization.ipynb
```

This notebook is used to inspect audio waveforms and ATAS event outputs.

## `Notebooks/ML_Analysis/`

These notebooks use the combined analysis dataset and run classification analyses.

# Notebook Workflow

## 1. Core ATAS Function Notebook

### `Scripts/ATAS_functions_CWS_CWNS.ipynb`

This notebook defines the main functions used by the automated ATAS processing pipeline.

It includes functions for:

- Loading audio files
- Bandpass filtering
- Noise reduction
- Audio normalization
- Audio trimming
- Speech/pause event detection
- Event-level statistics
- File-level summary statistics

This notebook is used by:

```text
Automated_ATAS_Streamlined_CWS_CWNS.ipynb
```

It does not produce final analysis files by itself.

---

## 2. Automated ATAS Processing

### `Automated_ATAS_Streamlined_CWS_CWNS.ipynb`

This is the main automated processing notebook.

It processes clipped audio files from:

```text
Audio_files_clipped/
```

using metadata files from:

```text
Stat_csv_files/CWNS_final.csv
Stat_csv_files/CWS_final.csv
```

The pipeline:

1. Reads the metadata CSV.
2. Extracts filenames from the `File_name` column.
3. Finds matching `.wav` files in `Audio_files_clipped/`.
4. Applies filtering, noise reduction, and normalization.
5. Detects speech and pause events.
6. Computes event-level and file-level statistics.
7. Saves participant-level event CSV files.
8. Saves group-level summary CSV files.

### Main Outputs

Participant-level event CSV files:

```text
Individual_OutputCSV_Files/
```

Group-level summary CSV files:

```text
Stat_csv_files/CWNS_All_files_together_50_ms_1_win_50_150_csv.csv
Stat_csv_files/CWS_All_files_together_50_ms_1_win_50_150_csv.csv
```

# Terminology

- **ATAS** — Automatic Temporal Analysis of Speech
- **CWS** — Children Who Stutter
- **CWNS** — Children Who Do Not Stutter
- **Pause event** — A detected non-vocal interval within the analyzed speech sample
- **Speech event** — A detected vocal interval within the analyzed speech sample
- **Long pause** — A pause category computed during the long/short pause analysis workflow (>150 ms)
- **Short pause** — A pause category computed during the long/short pause analysis workflow (=< 150 ms)
